# 5.3 myStudio Pro Introduction

**myStudio Pro** is a multi-functional robot programming and control software that provides users with a one-stop solution including visual programming interaction, quick motion control, drag teaching, robot status query and configuration. The software integrates five main functional modules: `Block Programming`, `Debug Panel`, `Resource Center`, `Scene Apps`, and `Configuration Center`, covering the full workflow from programming to debugging, from learning to deployment.

The **Block Programming** module draws inspiration in function and design philosophy from the children's programming language Scratch developed by the Massachusetts Institute of Technology, using a graphical block-splicing approach for programming. Users intuitively combine modules by dragging and dropping, gradually building complete code logic. The entire process is simple to operate and easy to understand, making it particularly suitable for programming beginners and educational scenarios.

From the user experience perspective, **Block Programming** is a low-barrier visual code generation tool that makes programming as easy and intuitive as building with blocks. From the developer's perspective, this module is essentially a text editor capable of dynamically generating structured code. The code generated by user drag-and-drop interactions is ultimately converted into executable instruction sequences on the robot. This design and interaction approach not only reduces the learning curve but also ensures the professionalism and executability of the programs.

**Debug Panel**

This module allows jog control of individual joint angles and coordinates, with configurable step sizes for each movement. It displays the robot arm posture in real time and allows manual control of the signal on/off status of corresponding IO ports.

**Resource Center**

This module provides users with convenient resource navigation, centrally displaying common external link entries including technical documentation, official contact information, and more. Users can quickly access relevant support materials without manual searching, improving usage and maintenance efficiency.

**Scene Apps**

This module provides two core robot functions: Writing & Drawing and Laser Engraving. Users can perform graphic editing and preview operations, which are converted into real control commands to realize the robot's writing, drawing, and laser engraving functions.

**Configuration Center**

This module covers basic configuration options for the software and robot system. Users can perform language switching, joint motion limit settings, system update checks, and software driver updates.

## Block Programming

`Block Programming` is a fully visual modular programming interface, a graphical programming language suitable for beginners to become familiar with programming. Users develop applications by dragging and dropping puzzle pieces, creating both simple and complex functions. It supports saving, loading, single-step debugging execution, and executing individual selected blocks.

## Alert Notifications

Function: Display robot error information. Left-click to open the error log window.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/home-12.png" />

If the robot encounters an error during operation, the application captures the exception and displays it in the error log interface. The error log table contains the following columns:

- number: Error log sequence number
- time: Time when the error occurred
- type: Type of error
- description: Error description information

When the application captures an error, it first shows a popup with a solution. If you don't want to handle the error, you can ignore it. After disconnecting and reconnecting the device, or entering the error log interface and clicking the "Clear" button, the popup will reappear and be saved to the error log table.

Taking a Joint 1 limit exceeded error as an example:

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/home-13.png" />

1. When a robot anomaly is captured, a specific warning popup appears. The popup consists of 4 parts: 1) The specific content of the current exception; 2) The solution for the current exception (displayed if resolvable/recoverable, otherwise no content); 3) The 'Repair' button for clearable/recoverable exceptions, which automatically repairs the exception when triggered (hidden if not applicable); 4) The confirmation button to ignore the current exception if you don't want to handle it.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/home-14.png" />

2. Resolved exceptions are displayed in the historical alarm table within the exception list, while currently existing exceptions are displayed in the current alarm table, with the exception duration automatically recorded.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/home-15.png" />

3. (1) Robot log download button -- automatically retrieves logs from the past day and downloads them as ".log" files to the corresponding storage location. If the exception is repairable, click the repair button (2) to perform the repair. Click the clear records button (3) to clear resolved historical alarm records.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/home-16.png" />
