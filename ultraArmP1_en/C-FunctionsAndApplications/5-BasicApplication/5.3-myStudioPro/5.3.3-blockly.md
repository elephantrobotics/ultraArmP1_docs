# Block Programming

You can click the `Block Programming` icon, or click the New File button to enter the Block Programming page.

> Alternatively, you can also click the `Open File` button to automatically jump to the Block Programming page, which will simultaneously load your saved file content including workspace, waypoints, and trajectory files (for how to save workspaces, [click here](./5.3.3-blockly.md#2-basic-function-usage)).<br />
> 
> Entering the Block Programming page by clicking the Block Programming icon or the New File button is equivalent to creating a new workspace.

## 1 Block Programming Main Interface Introduction

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/blockly-1.png" />

| ------ | ------------------------------------------------------------ |
| 1 | Save: Save block program files. By default, saves to File Management - Test File Space. |

| 3 | File Management: Open the file list panel for editing, deleting, and other operations on block program files. |

| 13 | Zoom In: Zoom in on the workspace. |
| 14 | Zoom Out: Zoom out on the workspace. |
| 15 | Trash Can: Drag blocks here to delete them, or retrieve deleted blocks from here. |

## 2 Basic Function Usage

Let's write a small case to introduce the basic usage of `Block Programming`.

**Case Description:** Control the robotic arm to return to zero, then move Joint 1 to 20 degrees, then return to zero again.

**Step 1:** First click the `Posture` button to open the posture view, where you can see the current posture of the simulated robotic arm.

- 1: Toggle posture panel: Shows when hidden, hides when shown.
- 2: Robot simulation model, simulates motion in real-time based on current actual angles.

**Step 2:** Start Programming

Open the Toolbox category `Motion Control`, select subcategory `Angle & Coordinate`, and drag the `Set All Angles` block to the workspace.

This block controls each joint of the robotic arm to move to the given angles, with a default speed of 20.

- Change `Joint 1` of the new block to 20;

## 7 Program Control

Introduces how to debug block code using the `Run Panel`, enabling program `Pause`, `Resume`, **`Single-Step Execution`**, and `Stop`.

Write a program to control the movement of the robotic arm's `Joint 1`.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/blockly-29.png" />

Click the `Run Panel` button. When the run panel pops up, click `Run All`, then immediately click the `Pause` button. The program will pause after executing the first instruction `mc.set_angle(1,0,20)`.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/blockly-30.png" />

The program is now paused. The next instruction to execute, `time.sleep(3)`, is displayed.

At this point:

- If you click the `Resume` button, the program will continue executing automatically;
- If you click the `Single-Step Execution` button, the program will execute the `time.sleep(3)` instruction. Clicking `Single-Step Execution` again will execute the next instruction, `mc.set_angle(1,-90,20)`;
- If you click the `Stop` button, the program will be terminated;

It's up to you to decide how to proceed!

## 9 Waypoints

This chapter introduces how to use the waypoint function to control the robotic arm.

Function Description: By entering `multiple waypoint data entries (joint and coordinate data of the robotic arm)`, quickly control the robotic arm to reach each waypoint position. Open the waypoint list page, long-press the end button to release the robotic arm, manually adjust the robotic arm posture, and click Add New Point to add a waypoint data entry.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/wp-1.png" />

| No. | Description |
| ------ | -------------------------------------------------------- |
| 1 | Click the "Import" button to import previously saved waypoint information JSON files. |
| 2 | Click the "Save" button to export all waypoint information into a single JSON file. |
| 3 | Click the "Delete" button to delete all selected waypoints from the waypoint list. |
| 4 | Click the "Copy" button to copy coordinate information of all currently selected waypoints. |
| 5 | Click the "Rename" button to rename all currently selected waypoints. |
| 6 | Click the "Overwrite Current Point" button to quickly overwrite coordinate information of all selected waypoints with the current robotic arm coordinate information. |
| 7 | List data selection column, functions 3-6 require this function. |
| 8 | Click the "Run" button to move the robotic arm to the currently recorded waypoint position. |
| 9 | Click the "Add New Point" button to record the current coordinate information of the robotic arm. |
| 10 | Click the "Generate Code" button to quickly generate waypoint blocks into the Block Programming workspace. |

In `Block Programming`, select waypoint blocks and add them to the programming workspace. Alternatively, use the `Generate Code` button on the waypoint list to append a generated waypoint block after the last block in the workspace.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/wp-2.png" />

Waypoint blocks are associated with data in the waypoint list. Select any waypoint from the list for association.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/wp-3.png" />

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/wp-4.png" />

Add new point information to the waypoint block. Select Add New Point, click the edit icon to open the add waypoint dialog. A custom waypoint name is generated by default. Use + or - to adjust waypoint information, then click the save waypoint button to add the waypoint data.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/wp-5.png" />

> Additionally, waypoint blocks can modify the motion mode. Joint motion mode: moves based on the current waypoint's joint information. Linear motion mode: moves based on the current waypoint's coordinate information.

| No. | Description |
| ---- | ------------------------------------------------------------ |
| 1 | Select a waypoint and click the edit icon to open the waypoint edit dialog. |
| 2 | Enter the waypoint name. |
| 3 | Display the robotic arm's joint and coordinate data. Adjust waypoint data using the left (-) and right (+) buttons. |
| 4 | Click save waypoint to add or modify waypoint data. |

## 10 Trajectory

Introduces how to use the trajectory function to control the robotic arm and achieve drag teaching.

Function Description: By `recording trajectory data (robotic arm joints)`, quickly control the robotic arm to reach each trajectory point position. Open the trajectory list page, click the Record Trajectory button to release all joints of the robotic arm. Press the end button and manually drag the robotic arm to record the trajectory. Click Stop Recording and wait for trajectory saving to complete recording. (For file-related operations, [click here](./5.3.3-blockly.md#11-file-management)).

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/wp-6.png" />

| No. | Description |
| ---- | ------------------------------------------------------------ |
| 1 | First select the trajectories to delete (at least one must be checked for the button to be available), then click delete to delete all checked trajectories. |
| 2 | List data selection column, function 1 requires this function. |
| 3 | Click the "Replay" button to replay the trajectory. |
| 4 | Click the "Copy" button to copy the current trajectory to the Test File workspace. |
| 5 | Click the "Record Trajectory" button. The robotic arm will prompt a secondary confirmation for trajectory recording. After confirmation, recording begins (max 120 seconds, auto-stops if exceeded). |

In `Block Programming`, select trajectory blocks and add them to the programming workspace. In the workspace, click the dropdown menu of the trajectory block to select data from the current trajectory list page.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/wp-7.png" />

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/wp-8.png" />

After selection, open the run panel and click Run All to replay the trajectory using block execution.

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/wp-9.png" />

After the program completes, the trajectory replay is finished. Of course, you can also combine with other blocks for more complex programming.

## 11 File Management

Introduces how to use the File Management module functionality.

Function Description: The File Management module makes it convenient to save, import, and export files from different file spaces. There are two ways to open the file list: Homepage - Open File or the myBlockly page File Operation menu - Open option, which opens the file management list and loads your saved file content including workspaces, waypoints, and trajectory files (for how to save workspaces, [click here](./5.3.3-blockly.md#2-basic-function-usage)).

<img width = "1200" align = "center" src="../../../resources/C-FunctionsAndApplications/5-BasicApplication/5.3-myStudioPro/file-1.png" />

| No. | Description |
| ------ | ------------------------------------------------------ |
| 1 | Test File Space: Switch to this option to paginate and query all test files. |
| 2 | Production File Space: Switch to this option to paginate and query all production files. |
| 3 | Recycle Bin File Space: Switch to this option to paginate and query all recycle bin files (files deleted from Test/Production space go to the recycle bin). |
| 4 | Search input box: Currently supports only fuzzy search by filename. Press Enter or click function 5 to search. |
| 5 | Search button: Click to perform a paginated query based on the input box content. |
| 6 | Import button: Click to import files from the current device. |
| 7 | Export button: Disabled by default, enabled when files are selected. Click to export selected files. |
| 8 | List data selection column, functions 7, 11-14 require this function. |
| 9 | Single file operation buttons, including: `Edit` - loads file content to workspace; `Preview` - applicable only for trajectory files, 3D model simulates trajectory point movement. |
| 10 | Pagination button: Enables pagination when the current space exceeds the specified limit. |
| 11 | Batch operation button, `Delete` - deletes selected files to the recycle bin. |
| 12 | Single file operation button: Selecting one file and clicking loads file content to workspace and automatically opens the debug panel. |
| 13 | Single file operation button: Selecting one file and clicking loads file content to workspace and automatically opens the debug panel, where you can set the file's loop count - infinite loop or count-based loop. |
| 14 | Batch operation button: Selecting one or more files and clicking publishes the files to the production space. |
