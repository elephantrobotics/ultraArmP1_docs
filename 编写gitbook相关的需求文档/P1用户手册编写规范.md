### 命名规则

| 层级 | 规则 | 示例 |
|------|------|------|
| 一级目录 | `数字-英文名`（与 450 一致） | `1-ProductInformation`、`2-BasicSettings` |
| 二级目录 | `数字.英文名` | `1-ProductIntroduction`、`3-UserNotes` |
| 叶子文件 | `编号-英文名.md` | `3.1-SafetyInstruction.md`、`4.1-ProductStandardList.md` |
| 章节入口 | `README.md` | 每章/节统一使用 README.md |
| 多子页章节 | 独立目录，内含 README + 子页 | `5.2-minirobot/`、`6.1-python/` |

> **注意**：P1 当前使用 `A-`、`B-` 前缀，为与 450 保持一致，建议后续统一为数字前缀。若领导确认沿用字母前缀，则仅需替换前缀部分。

## 三、内容编写规范（严格参照 450）

### 3.1 文件结构模板

每篇文档固定包含三部分：

```markdown
# 章节标题（一级标题，对应 SUMMARY.md 中的显示名）

## 1. 第一小节
正文内容...

## 2. 第二小节
正文内容...

---

[← 上一章](...路径) | [下一章 →](...路径)
```

### 3.2 标题层级

| 层级 | 语法 | 用途 |
|------|------|------|
| `#` | 一级标题 | 页面标题（每页仅一个） |
| `##` | 二级标题 | 大节（1, 2, 3...） |
| `###` | 三级标题 | 小节（1.1, 1.2...） |
| `#####` | 五级标题 | 特殊强调文字（如产品名） |

> **注意**：450 中不使用 `####`（四级标题），跳级使用 `#####` 作突出显示。

### 3.3 图片规范

```markdown
<img src = ../../resources/对应目录/图片名.png
width = "1200" align = "center">
```

| 规则 | 说明 |
|------|------|
| 路径 | 使用相对路径 `../../resources/...`，镜像 docs 目录结构 |
| 宽度 | 统一 `"1200"`（全宽）或 `"800"`（接口特写） |
| 对齐 | 统一 `align = "center"` |
| 格式 | PNG 优先 |
| 产品展示图 | 建议 16:9 比例 |

### 3.4 表格规范

```markdown
| 列A | 列B |
| :--- | :--- |
| 内容 | 内容 |
```

- 使用 Markdown 原生表格
- 表头左对齐 `:---`
- 复杂数据可拆分为多个小表

### 3.5 列表规范

```markdown
- 无序项1
- 无序项2

1. 有序项1
2. 有序项2
```

### 3.6 强调规范

- 关键术语、产品名、重要提示：`**粗体**`
- 禁止使用斜体作为强调（450 中不使用 `*斜体*`）

### 3.7 分隔与导航

- 大节之间使用 `---` 水平线分隔
- 每页末尾固定添加导航链接：
  ```markdown
  [← 上一章](../路径) | [下一章 →](../路径)
  ```

### 3.8 警告/提示样式

参照 450 使用简单的图标+表格形式：

```markdown
| ![danger](../../resources/.../warning.png) |
| :--- |
| 具体危险说明文字 |
```

## 四、SUMMARY.md 编写规范

### 4.1 格式

```markdown
# Summary

* [Introduction](README.md)

## 一级分类名
* [X 章节名](路径/README.md)
    * [X.X 小节名](路径/文件名.md)
        * [X.X.X 子页](路径/文件名.md)
```

- 一级分类使用 `##` 标题分组
- 每个缩进级别用 4 个空格
- 显示名称用中文（中文版）或英文（英文版）
- 链接路径为相对于 SUMMARY.md 的路径

### 4.2 目录深度

显示到第三级（如 `5.2.1`），更深内容通过页面内部导航处理。

## 五、中英文双版同步规范

| 规则 | 说明 |
|------|------|
| 目录结构 | 中英文版完全一致 |
| 文件名 | 统一使用英文命名，中文名仅在 SUMMARY.md 显示文字中体现 |
| README.md | 中英文各写各的 |
| resources | 共用同一套图片资源（无文字或使用英文标注） |

## 六、编写流程

1. 先在 `ultraArmP1_cn` 中编写中文版
2. 按 450 对应章节的结构逐章填充内容
3. 中文版定稿后翻译为英文版放入 `ultraArmP1_en`
4. 更新两侧 `SUMMARY.md` 保持同步
5. 本地 `gitbook build` 验证渲染效果
6. 提交到 `develop` 分支推送
